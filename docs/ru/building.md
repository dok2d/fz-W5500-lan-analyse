# Сборка из исходников

## Требования

- [ufbt](https://github.com/flipperdevices/flipperzero-ufbt) (micro Flipper Build Tool)

Установка ufbt:

```bash
pip install ufbt
```

## Команды сборки

Все команды выполняются из корневой директории проекта:

```bash
ufbt build              # компиляция .fap
ufbt launch             # сборка и запуск на подключённом Flipper через USB
ufbt install            # сборка и установка .fap на SD-карту Flipper
ufbt format             # автоформатирование кода clang-format
ufbt lint               # проверка форматирования без изменения файлов
```

## Результат сборки

Скомпилированный `.fap` файл появляется в `dist/`. Его можно скопировать вручную на SD-карту Flipper:

```
/ext/apps/GPIO/lan_tester.fap
```

## Ручная установка

1. Соберите `.fap` командой `ufbt build`
2. Подключите Flipper Zero по USB
3. Откройте qFlipper или смонтируйте SD-карту
4. Скопируйте `dist/lan_tester.fap` в `/ext/apps/GPIO/` на SD-карте
5. Перезагрузите Flipper или перейдите в меню **GPIO**

Альтернативно, `ufbt launch` делает всё это автоматически и запускает приложение.

## Совместимость с прошивкой

Приложение собирается под **Flipper Zero Official Firmware (OFW)**. Совместимость с кастомными прошивками (Unleashed, Momentum и др.) не гарантируется, но обычно работает, если прошивка близко следует OFW API.

Версия SDK управляется автоматически через ufbt -- он скачивает подходящий SDK при первой сборке.

## CI/CD

Проект использует GitHub Actions для автоматических сборок:

### CI (Pull Requests)

Определён в `.github/workflows/ci.yml`. При каждом PR в `main`:

- Сборка приложения через `ufbt build`
- Проверка форматирования через `ufbt lint`
- PR, меняющие только документацию (`*.md`, `docs/**`, `LICENSE`), пропускают CI

### Release (Теги)

Определён в `.github/workflows/release.yml`. При пуше тега версии (`v*.*.*`):

- Сборка `.fap` бинарника
- Извлечение release notes из `CHANGELOG.md` для данной версии
- Создание GitHub Release с приложенным `.fap` файлом

## Рабочий процесс разработки

1. Создайте feature-ветку: `git checkout -b feat/my-feature`
2. Внесите изменения в проект
3. Отформатируйте код: `ufbt format`
4. Соберите и протестируйте: `ufbt build` или `ufbt launch`
5. Обновите документацию при необходимости (см. [CONTRIBUTING.md](../../CONTRIBUTING.md))
6. Откройте PR в `main`

### Стиль кода

- **Язык**: C99
- **Именование**: `snake_case` для функций/переменных, `UPPER_CASE` для макросов/констант
- **Отступы**: 4 пробела, без табов
- **Скобки**: K&R стиль (открывающая скобка на той же строке)
- **Форматирование**: контролируется `.clang-format` в корне проекта; запустите `ufbt format` перед коммитом

### Вендорная библиотека

Директория `lib/ioLibrary_Driver/` содержит вендорную копию драйвера WIZnet W5500. **Не изменяйте** файлы в этой директории. Компилируются только следующие компоненты:

- `Ethernet/W5500/*.c` -- драйвер чипа W5500
- `Ethernet/*.c` -- абстракция сокетов
- `Internet/DHCP/*.c` -- DHCP-клиент
- `Internet/DNS/*.c` -- DNS-резольвер
- `Internet/ICMP/*.c` -- ICMP ping и traceroute
