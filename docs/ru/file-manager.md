# Файловый менеджер

Файловый менеджер запускает HTTP-сервер на Flipper, позволяя управлять файлами на microSD-карте из любого веб-браузера в той же сети.

## Как это работает

1. Flipper получает IP-адрес по DHCP
2. HTTP-сервер запускается на **порту 80**
3. Генерируется случайный токен авторизации
4. URL с токеном отображается на экране Flipper
5. Откройте URL в любом браузере для управления файлами

## Авторизация

Каждая сессия генерирует случайный 4-символьный hex-токен (напр. `a3f1`). URL на экране Flipper включает этот токен:

```
http://192.168.1.42/?t=a3f1
```

**Все HTTP-запросы должны содержать параметр `?t=XXXX`.** Запросы без токена получают `403 Forbidden`. Токен автоматически включён во все ссылки веб-интерфейса, так что вводить его нужно только один раз при открытии начального URL.

Токен виден только на физическом экране Flipper -- он не обнаруживается по сети. Это предотвращает несанкционированный доступ с других устройств в LAN.

## Использование

1. Откройте **Tools → File Manager** на Flipper
2. Дождитесь получения IP по DHCP (показан на экране)
3. Запомните URL с токеном на экране Flipper
4. Откройте URL в браузере на любом устройстве в той же сети
5. Просматривайте, загружайте, скачивайте, удаляйте файлы
6. Нажмите **Back** на Flipper для остановки сервера

## Операции

### Просмотр каталогов

Навигация по директориям кликом на имена папок. Текущий путь показан вверху. Ссылка `..` ведёт в родительский каталог.

Для каждой записи показывается:
- Имя файла/папки
- Размер файла (для файлов)
- Ссылки действий (скачать, удалить)

### Скачивание файлов

Кликните ссылку **download** рядом с любым файлом. Браузер скачает файл с оригинальным именем.

Большие файлы передаются порциями -- Flipper читает и отправляет данные блоками в пределах ограничений памяти.

### Загрузка файлов

На каждой странице каталога внизу есть форма **Upload**. Выберите файл и нажмите Upload. Файл загружается через POST-запрос `multipart/form-data`.

**Ограничения:**
- Скорость загрузки ограничена пропускной способностью SPI (~300-500 КБ/с)
- Очень большие файлы могут загружаться долго
- Только один файл за раз

### Создание папок

Введите имя папки в форме **Create Folder** и отправьте. Папка создаётся в текущем каталоге.

### Удаление

Кликните **Delete** рядом с файлом или папкой. Появится диалог подтверждения. Папки должны быть пустыми для удаления.

## Веб-интерфейс

Интерфейс использует тёмную тему со встроенной алфавитной сортировкой (папки первыми). Работает на десктопных и мобильных браузерах.

### Кастомные темы

Вы можете заменить стандартные CSS и/или JavaScript, разместив файлы на SD-карте:

- **apps_data/lan_tester/web/custom.css** -- заменяет стандартные стили
- **apps_data/lan_tester/web/custom.js** -- заменяет стандартный скрипт сортировки

Файлы обнаруживаются при запуске File Manager. Для применения изменений перезапустите инструмент (Back, затем откройте снова).

Примеры тем и расширенный custom.js находятся в репозитории в **docs/filemanager_themes/**:

**Темы (custom.css):**
- **hacker-green.css** -- зелёный терминал на чёрном, свечение текста
- **arctic-light.css** -- светлая минималистичная тема с синими акцентами
- **cyberpunk.css** -- неоновый розовый на тёмно-фиолетовом
- **solarized-dark.css** -- классическая палитра Solarized

**Расширенный JS (custom.js):**
- Сортировка по Name или Size кликом на заголовок
- Поиск/фильтрация в реальном времени
- Кликабельные хлебные крошки в пути
- Мультивыбор с чекбоксами и пакетное удаление
- Drag-and-drop загрузка файлов
- Просмотр файлов: текст, изображения, аудио, видео, PDF, hex-дамп бинарных

Для использования: скопируйте нужный файл в **apps_data/lan_tester/web/** на SD-карте и переименуйте в **custom.css** или **custom.js**.

## Технические детали

### TCP Send Layer

Функция `send()` W5500 неблокирующая и может не отправить все данные сразу. Файловый менеджер реализует надёжный TCP send слой с опросом `SEND_OK`:

1. Запись данных в TX-буфер W5500
2. Команда SEND
3. Опрос статуса SEND_OK
4. Повтор до отправки всех данных

Это обеспечивает надёжную доставку HTTP-ответов, особенно при скачивании больших файлов.

### Обработка соединений

- Один клиент за раз (один TCP-сокет)
- Keep-alive соединения поддерживаются для ускорения последовательных запросов
- Сокет принудительно закрывается при выходе для предотвращения зависания (функция `send()` WIZnet имеет внутренний `while(1)` без таймаута)
