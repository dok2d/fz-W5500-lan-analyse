# PXE Server

PXE-сервер превращает Flipper Zero в минимальный сервер сетевой загрузки. Он может загружать компьютеры, серверы и встраиваемые устройства по Ethernet через стандартные протоколы DHCP + TFTP.

## Что такое PXE-загрузка?

PXE (Preboot Execution Environment) позволяет компьютеру загружать ОС по сети вместо локального диска. Процесс загрузки:

1. Целевая машина отправляет DHCP-запрос с PXE-расширениями
2. DHCP-сервер отвечает IP-адресом и именем boot-файла
3. Целевая машина скачивает boot-файл по TFTP
4. Boot-файл загружается и запускает ОС или загрузчик

Flipper может выступать и DHCP-сервером, и TFTP-сервером одновременно, или только TFTP, если в сети уже есть внешний DHCP.

## Поддерживаемые форматы boot-файлов

| Расширение | Тип | Описание |
|-----------|------|----------|
| `.kpxe` | Legacy BIOS | iPXE/gPXE UNDI загрузчик |
| `.efi` | UEFI | EFI загрузчик (x64, ARM) |
| `.pxe` | Legacy BIOS | Стандартная PXE-программа |
| `.0` | Legacy BIOS | Универсальный PXE-файл (напр. `pxelinux.0`) |

## Подготовка

### 1. Разместите boot-файлы на SD-карте

Скопируйте boot-файлы на SD-карту Flipper:

```
/ext/apps_data/lan_tester/pxe/
├── undionly.kpxe        # Legacy BIOS
├── ipxe.efi             # UEFI
└── (другие boot-файлы)
```

**Рекомендуется**: скачайте `undionly.kpxe` (~70 КБ) с [netboot.xyz](https://boot.netboot.xyz). Этот iPXE-загрузчик предоставляет меню для загрузки различных ОС и утилит через интернет.

### 2. Подключите к целевой машине

Подключите Ethernet-порт Flipper к целевой машине:

- **Прямой кабель**: Flipper ↔ целевая машина (любой тип кабеля, Auto-MDI/MDI-X)
- **Через свитч**: Flipper и целевая машина на одном свитче/VLAN

### 3. Настройте и запустите

Откройте **Tools → PXE Server** на Flipper. Появится экран настроек.

## Автоопределение DHCP

При входе в настройки PXE приложение автоматически зондирует сеть:

1. Отправляет DHCP Discover
2. Ждёт 5 секунд

**Если внешний DHCP-сервер ответил:**
- Встроенный DHCP отключается (два DHCP-сервера -- это проблема!)
- Server IP, Client IP и Subnet заполняются из обнаруженной сети
- Flipper работает как **только TFTP** сервер

**Если DHCP-сервер не ответил:**
- Встроенный DHCP включается
- Подсеть по умолчанию: 192.168.77.0/24
- Server IP: 192.168.77.1
- Client IP: 192.168.77.10
- Flipper предоставляет **DHCP + TFTP**

Все поля IP редактируемы -- автоопределённые значения можно изменить.

## Экран настроек

| Пункт | Описание |
|-------|----------|
| **Start PXE** | Запуск сервера |
| **DHCP Server** | ON/OFF -- встроенный DHCP-сервер |
| **Boot File** | Выбор из найденных файлов (стрелки влево/вправо) |
| **Server IP** | IP Flipper в PXE-сети |
| **Client IP** | IP для загружаемой машины |
| **Subnet Mask** | Маска подсети |
| **Help** | Краткая справка |

### Выбор boot-файла

Если в PXE-директории найдено несколько boot-файлов (до 8), используйте **стрелки влево/вправо** для переключения. Файлы отсортированы с приоритетом:

1. `undionly.kpxe`
2. `ipxe.efi`
3. `snponly.efi`
4. Остальные файлы по алфавиту

## Настройка BIOS целевой машины

На целевой машине должна быть включена PXE/сетевая загрузка:

1. Войдите в BIOS/UEFI (обычно F2, F12, Del или Esc при POST)
2. Найдите настройки **Boot** или **Network Boot**
3. Включите **PXE Boot** или **Network Boot**
4. Установите приоритет загрузки **Network** первым (или используйте одноразовое boot-меню, обычно F12)
5. Для UEFI-загрузки убедитесь, что режим соответствует файлу (UEFI для `.efi`, Legacy для `.kpxe`)

## Индикация прогресса

Во время раздачи boot-файла экран показывает:

- Текущее состояние передачи (ожидание, отправка, завершено)
- Счётчик блоков (текущий / всего)
- Прогрессбар
- Скорость передачи

После завершения передачи сервер сбрасывается в ожидание следующего клиента. Несколько машин можно загрузить последовательно (не одновременно).

## Решение проблем

| Проблема | Решение |
|----------|---------|
| Целевая не начинает PXE-загрузку | Включите Network Boot в BIOS. Проверьте порядок загрузки. |
| "No boot file found" | Поместите .kpxe/.efi файлы в `/ext/apps_data/lan_tester/pxe/` |
| DHCP не доходит до целевой | Проверьте кабель. Если на свитче с DHCP, отключите DHCP Flipper. |
| Передача зависает | Попробуйте другой/короче кабель. SPI может ограничивать скорость. |
| UEFI-машина получает BIOS-файл | Выберите правильный `.efi` boot-файл в настройках. |
