# Безопасность

## Обзор

LAN Tester -- инструмент диагностики локальной сети, а не интернет-сервис. Меры безопасности направлены на защиту от угроз в общей LAN: несанкционированный доступ, атаки path traversal из формированных запросов, вредоносные сетевые ответы и межсайтовый скриптинг.

Приложение **не** реализует TLS/HTTPS -- в FAP SDK Flipper Zero нет криптографических библиотек. Вся сетевая коммуникация не зашифрована. Используйте только в доверенных сетях.

## Меры защиты по компонентам

### Файловый менеджер (HTTP-сервер)

Наиболее чувствительный к безопасности компонент, так как предоставляет HTTP-сервер, принимающий запросы из сети.

**Токен авторизации**: каждая сессия генерирует случайный 4-символьный hex-токен через аппаратный ГСЧ. Токен отображается на экране Flipper и требуется в каждом HTTP-запросе как параметр `?t=XXXX`. Запросы без токена получают `403 Forbidden`. Токен не обнаруживается по сети -- для его чтения нужен физический доступ к Flipper.

**Защита от path traversal**: все URL-пути проверяются перед файловыми операциями. Последовательности `..` отклоняются с `403 Forbidden`. Это предотвращает запросы вида `GET /../../../etc/passwd` от выхода за пределы корня SD-карты.

**Предотвращение XSS**: имена файлов экранируются в HTML (замена `<`, `>`, `&`, `"`, `'` на HTML entities) перед отображением в листингах каталогов. Это предотвращает инъекцию JavaScript через вредоносно названный файл.

**Санитизация имён загрузок**: символы-разделители путей (`/` и `\`) удаляются из имён загружаемых файлов. Это предотвращает запись файла по пути вне текущей директории.

**Безопасность Content-Disposition**: спецсимволы удаляются из имён файлов в HTTP-заголовке `Content-Disposition` при скачивании. Это предотвращает атаки инъекции заголовков.

### PXE Server (TFTP)

**Защита от path traversal**: имена файлов TFTP, содержащие `..` или начинающиеся с `/`, отклоняются. Это предотвращает запрос вредоносным PXE-клиентом файлов за пределами директории `/ext/apps_data/lan_tester/pxe/`.

### DNS

**Валидация ответов**: DNS-ответы проверяются на соответствие IP-адреса источника ожидаемому DNS-серверу. Это снижает риск DNS-спуфинга от других устройств в LAN. Если IP источника не совпадает с сервером, которому был отправлен запрос, ответ отбрасывается.

### mDNS

**Ограничение глубины рекурсивных указателей**: mDNS-ответы могут содержать DNS-указатели сжатия имён, ссылающиеся на другие части пакета. Вредоносный ответ может создать рекурсивные указатели для переполнения стека. Парсер ограничивает следование указателям глубиной 4, предотвращая эту атаку.

### MAC-адрес

**Уникальный для устройства**: при первом запуске приложение генерирует случайный locally-administered MAC-адрес через аппаратный ГСЧ Flipper. Этот MAC сохраняется в `mac.conf` на SD-карте и переиспользуется при последующих запусках. Два устройства не получат одинаковый MAC по умолчанию.

## Что НЕ защищено

- **Нет шифрования**: весь сетевой трафик открыт. HTTP, DHCP, TFTP, DNS -- всё без шифрования.
- **Нет взаимной аутентификации**: токен File Manager аутентифицирует только браузер перед Flipper, не наоборот.
- **Нет ограничения скорости запросов**: HTTP-сервер не ограничивает частоту запросов. Поток запросов может замедлить Flipper.
- **Нет контроля доступа**: любой в LAN, знающий токен, может получить доступ ко всем файлам на SD.
- **Нет аутентификации TFTP**: PXE/TFTP не имеет механизма аутентификации -- любое устройство в LAN может запросить boot-файлы.
