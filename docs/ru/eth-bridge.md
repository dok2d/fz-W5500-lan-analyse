# ETH Bridge

ETH Bridge превращает Flipper Zero в USB-Ethernet сетевой адаптер через протокол CDC-ECM (Communication Device Class -- Ethernet Control Model). Телефон или ПК, подключённый к Flipper по USB, получает прозрачный доступ к LAN на уровне L2 через Ethernet-порт W5500.

## Как это работает

Мост работает на уровне L2 (Ethernet-фреймы), а не L3 (IP-пакеты). Это означает:

- Подключённый хост получает свой IP от DHCP-сервера сети
- Все протоколы работают прозрачно (HTTP, SSH, DNS и т.д.)
- Flipper действует как невидимый провод между USB и Ethernet
- Нет NAT, нет маршрутизации, нет настройки IP на Flipper

### Поток данных

```
Телефон/ПК  ←→  USB (CDC-ECM)  ←→  Flipper  ←→  W5500 (SPI)  ←→  LAN (RJ45)
```

Фреймы, полученные по USB, пересылаются в MACRAW-сокет W5500. Фреймы, полученные от W5500, пересылаются в USB. CPU Flipper обеспечивает копирование фреймов между двумя интерфейсами.

## Использование

1. Подключите модуль W5500 и Ethernet-кабель
2. Откройте **Tools → ETH Bridge** на Flipper
3. Flipper переключает USB-профиль с CDC Serial на CDC-ECM
4. Подключите телефон или ПК к Flipper по USB
5. Хост обнаруживает новый сетевой интерфейс и запрашивает IP по DHCP
6. Сетевой трафик идёт через мост

### Экранная информация

Во время работы моста на экране отображается:

- **USB**: состояние подключения (Connected / Not connected)
- **LAN**: статус линка, скорость, дуплекс
- **USB → LAN**: счётчик фреймов (из USB в Ethernet)
- **LAN → USB**: счётчик фреймов (из Ethernet в USB)
- **Errors**: счётчик ошибок SPI/USB

### Управление

- **OK**: включить/выключить запись PCAP
- **Back**: остановить мост и восстановить USB

## Запись PCAP

Нажмите **OK** во время работы моста для начала записи всего проходящего трафика в `.pcap` файл на SD-карте.

- Файлы сохраняются в `/ext/apps_data/lan_tester/pcap/`
- Формат имени: `bridge_YYYYMMDD_HHMMSS.pcap`
- Совместимы с Wireshark, tcpdump и другими PCAP-анализаторами
- Нажмите **OK** ещё раз для остановки записи

PCAP-файл захватывает фреймы из **обоих направлений** (USB→LAN и LAN→USB), давая полную картину трафика через мост.

**Примечание**: запись на SD во время работы моста добавляет задержку из-за операций записи. Для максимальной пропускной способности работайте без записи.

## Совместимость с ОС

| Платформа | Поддержка | Примечания |
|-----------|-----------|------------|
| **Linux** | Нативная | CDC-ECM распознаётся автоматически, драйверы не нужны |
| **macOS** | Нативная | CDC-ECM распознаётся автоматически |
| **Android** | Работает | Большинство устройств с USB OTG поддерживают CDC-ECM |
| **Windows** | Ограниченная | Нет встроенного CDC-ECM драйвера. Может работать со сторонними RNDIS/ECM драйверами |
| **iOS** | Не поддерживается | iOS не поддерживает USB Ethernet через CDC-ECM |

### Быстрый старт на Linux

```bash
# Интерфейс появляется автоматически (напр. usb0 или enp0sXXX)
# Получите IP по DHCP:
sudo dhclient usb0

# Или через NetworkManager:
nmcli device connect usb0
```

### Android

Подключите через USB OTG кабель или адаптер. Большинство Android-устройств с 2018+ поддерживают CDC-ECM нативно. Сетевое подключение появляется в Настройках → Сеть автоматически.

## Управление USB-профилем

При запуске моста Flipper сохраняет текущий USB-профиль (обычно CDC Serial для CLI/qFlipper) и переключается на CDC-ECM. При нажатии **Back**:

1. Мост останавливается
2. USB переключается обратно на исходный профиль
3. CLI и qFlipper снова доступны

Если приложение аварийно завершится или Flipper перезагрузится, USB-профиль может остаться в CDC-ECM. Перезагрузка Flipper восстановит профиль по умолчанию.

## Производительность

Основное узкое место -- шина SPI между STM32 Flipper и W5500:

- Тактовая частота SPI: ~8 МГц
- Теоретический максимум: ~1 МБ/с
- Практическая пропускная способность: 300-500 КБ/с в зависимости от размера фреймов

Этого достаточно для веб-сёрфинга, SSH, передачи файлов и обычной сетевой активности. Насытить 100 Мбит/с Ethernet-линк не получится.

## Сценарии использования

- **Дать телефону доступ в LAN**: подключить Android к проводной сети через Flipper
- **Сетевая диагностика**: дать ноутбуку доступ к сегменту сети, где есть только проводные порты
- **Захват трафика**: записать PCAP-дамп трафика через мост для анализа в Wireshark
- **Портативный сетевой tap**: увидеть, какой трафик генерирует устройство, пропустив его через Flipper с записью PCAP
