# Flipper Zero LAN Тестер (W5500) — Полная документация

Превратите **Flipper Zero + модуль W5500 Lite** в профессиональный портативный LAN-тестер.

---

## Возможности

| Функция | Описание |
|---|---|
| **Link Info** | Статус PHY-линка, скорость (10/100 Мбит/с), дуплекс, MAC-адрес, версия W5500 |
| **DHCP Analyzer** | Анализ Discover/Offer (IP не берётся!), фингерпринтинг опций |
| **ARP Scanner** | Сканирование подсети с определением вендора по OUI (~120 производителей) |
| **Ping** | Echo Request/Reply на любой IP с измерением RTT (4 пинга, таймаут 3с) |
| **Continuous Ping** | Графики RTT в реальном времени с min/max/avg и процентом потерь |
| **DNS Lookup** | Разрешение имён через UDP DNS из DHCP |
| **Traceroute** | ICMP-трассировка маршрута с RTT на каждый хоп (до 30) |
| **Ping Sweep** | ICMP-сканирование всей подсети, CIDR автоопределяется из DHCP |
| **Port Scanner** | TCP connect-сканирование: Top-20 (быстро) и Top-100 (полно) |
| **LLDP/CDP** | Пассивное обнаружение соседей IEEE 802.1AB и Cisco CDP |
| **mDNS/SSDP** | Обнаружение сервисов через multicast DNS и UPnP/SSDP |
| **STP/VLAN** | Пассивный захват BPDU + определение 802.1Q VLAN-тегов |
| **Статистика** | Счётчики фреймов по типу и EtherType |
| **Wake-on-LAN** | Отправка magic-пакетов на любой MAC-адрес |
| **ETH Bridge** | USB-Ethernet мост через CDC-ECM, опциональный PCAP-дамп на SD |
| **PXE Server** | PXE-сервер с автодетектом DHCP + TFTP, выбор boot-файла |
| **File Manager** | Веб-менеджер файлов SD через HTTP с токеном авторизации |
| **История** | Автосохранение результатов с метками времени |
| **Настройки** | Автосохранение, звук/вибрация, очистка истории, MAC Changer |

## Оборудование

### Необходимо

- **Flipper Zero** (официальная прошивка OFW)
- **W5500 Lite** Ethernet-модуль (или любая плата на W5500 с SPI)

### Где купить

- [W5500 Ethernet модуль для Flipper Zero](https://flipperaddons.com/product/w5500-ethernet/)

### Подключение

```
Модуль W5500     GPIO Flipper Zero
-----            -----
MOSI (MO)    ->  A7  (пин 2)
SCLK (SCK)   ->  B3  (пин 5)
CS   (nSS)   ->  A4  (пин 4)
MISO (MI)    ->  A6  (пин 3)
RESET (RST)  ->  C3  (пин 7)
3V3  (VCC)   ->  3V3 (пин 9)
GND  (G)     ->  GND (пин 8 или 11)
```

W5500 питается через OTG 3.3В Flipper'а, который включается автоматически при запуске.

## Сборка

### Требования

- [ufbt](https://github.com/flipperdevices/flipperzero-ufbt) (micro Flipper Build Tool)

### Сборка и установка

```bash
cd eth_tester
ufbt build              # только сборка
ufbt launch             # сборка и запуск на Flipper через USB
ufbt install            # установка .fap на SD-карту Flipper
```

Скомпилированный `.fap` файл появится в `dist/`. Его также можно скопировать вручную в `/ext/apps/GPIO/` на SD-карте Flipper'а.

## Архитектура

```
eth_tester/
├── application.fam              # Манифест FAP
├── eth_tester_app.c/h           # Точка входа, ViewDispatcher, логика
├── hal/w5500_hal.c/h            # SPI, GPIO, управление MACRAW-сокетом
├── usb_eth/                     # USB CDC-ECM сетевое устройство
├── bridge/                      # L2-пересылка фреймов + PCAP-дамп
├── protocols/                   # 16 модулей сетевых протоколов
├── utils/                       # OUI-поиск, парсинг пакетов
├── assets/icon.png              # Иконка FAP
└── lib/ioLibrary_Driver/        # Драйвер WIZnet W5500 (вендорная копия)
```

### Распределение сокетов W5500

| Сокет | Буфер | Использование |
|-------|-------|---------------|
| 0 | 8КБ RX/TX | MACRAW (захват всех фреймов) |
| 1 | 2КБ | DHCP-клиент |
| 2 | 2КБ | ICMP ping / traceroute (IPRAW) |
| 3 | 2КБ | DNS / WoL / mDNS / SSDP / File Manager HTTP |
| 4 | 1КБ | PXE TFTP listen (порт 69) |
| 5 | 1КБ | PXE TFTP передача данных |

### Потоки

- **Основной поток** (4КБ стек): GUI event loop через ViewDispatcher
- **Worker thread** (8КБ стек): создаётся по требованию для длительных операций
- Отмена: `volatile bool worker_running`, кнопка Back останавливает worker

### Модель памяти

- Буфер фреймов (1600Б) в куче, разделяется с worker thread
- Результаты сканирования в куче (ARP hosts, записи каталогов)
- Текстовые буферы через FuriString (динамическое выделение)
- Стек приложения 4КБ — все большие буферы в куче

## Руководство по использованию

### Начало работы

1. Подключите W5500 к Flipper Zero по схеме выше
2. Вставьте Ethernet-кабель в RJ45 разъём
3. Откройте **GPIO -> LAN Tester** на Flipper'е
4. В заголовке меню отображается статус линка: `LAN [UP 100M FD]`

### Network Info

- **Link Info** — статус линка, скорость, дуплекс, MAC. Используйте первым для проверки.
- **DHCP Analyze** — отправляет Discover, парсит Offer. IP не занимает. Показывает: предложенный IP, сервер, подсеть, шлюз, DNS, NTP, домен, время аренды, фингерпринт опций.
- **Statistics** — захват всех фреймов 10 секунд. Разбивка по типу назначения и EtherType.

### Discovery

- **ARP Scan** — сначала DHCP, затем сканирование подсети. IP/MAC/вендор для каждого хоста. Пакетная отправка (16 ARP за 15мс).
- **Ping Sweep** — ICMP-свип по CIDR. Автоопределение из DHCP или ручной ввод (`192.168.1.0/24`).
- **LLDP/CDP** — пассивное прослушивание до 60 секунд. Имя системы, порт, описание, IP управления, VLAN.
- **mDNS/SSDP** — запрос `_services._dns-sd._udp.local` и SSDP M-SEARCH. Сбор ответов ~10 секунд.
- **STP/VLAN** — прослушивание BPDU 30 секунд (STP/RSTP/MSTP). Определение 802.1Q VLAN-тегов.

### Diagnostics

- **Ping** — 4 ICMP echo с таймаутом 3с. По умолчанию — шлюз из DHCP.
- **Continuous Ping** — живой график RTT (128px), текущий/средний RTT и % потерь. До нажатия Back.
- **DNS Lookup** — разрешение имени (напр. `google.com`) через DNS из DHCP.
- **Traceroute** — ICMP с инкрементом TTL (1-30). IP и RTT каждого хопа.
- **Port Scan** — TCP connect. Top-20: 18 популярных портов. Top-100: 100 портов.

### Tools

- **Wake-on-LAN** — ввод MAC, отправка magic-пакета через broadcast UDP на порт 9.
- **ETH Bridge** — см. [ETH Bridge](#eth-bridge) ниже.
- **PXE Server** — см. [PXE Server](#pxe-server) ниже.
- **File Manager** — см. [Файловый менеджер](#файловый-менеджер) ниже.

### Settings

- **Auto-save results** — вкл/выкл автосохранение в историю.
- **Sound & vibro** — зелёный LED + вибро при успехе, красный при ошибке.
- **Clear History** — удалить все сохранённые файлы.
- **MAC Changer** — случайный или ручной MAC, сохраняется на SD.

## PXE Server

PXE-сервер превращает Flipper в сервер сетевой загрузки.

### Подготовка

1. Поместите boot-файлы на SD-карту: `/ext/apps_data/eth_tester/pxe/`
   - Поддерживаются: `.kpxe` (Legacy BIOS), `.efi` (UEFI), `.pxe`, `.0`
   - Рекомендуется: `undionly.kpxe` (~70КБ) с [netboot.xyz](https://boot.netboot.xyz)
2. Подключите Flipper к целевой машине через Ethernet
3. Откройте **Tools -> PXE Server**

### Автоопределение DHCP

При входе в настройки PXE приложение автоматически зондирует сеть (отправляет DHCP Discover, ждёт 5 секунд):

- **Внешний DHCP найден**: собственный DHCP выключается, поля Server IP / Client IP / Subnet заполняются из обнаруженной сети. Flipper работает только как TFTP-сервер.
- **Внешний DHCP не найден**: собственный DHCP включён с дефолтами (подсеть 192.168.77.x). Flipper раздаёт IP и файлы.

Все поля IP редактируемы — автозначения можно изменить.

### Выбор boot-файла

Если найдено несколько boot-файлов, используйте стрелки влево/вправо на пункте "Boot File" для переключения (до 8 файлов). Приоритетные файлы (`undionly.kpxe`, `ipxe.efi`, `snponly.efi`) показываются первыми.

### Порядок настроек

1. **Start PXE** — запуск сервера
2. **DHCP Server** — вкл/выкл
3. **Boot File** — выбор из найденных файлов
4. **Server IP** — IP Flipper'а в PXE-сети
5. **Client IP** — IP для загружаемой машины
6. **Subnet Mask**
7. **Help**

### BIOS целевой машины

Включите Network/PXE Boot в настройках BIOS/UEFI. Установите приоритет загрузки с сети.

## Файловый менеджер

File Manager запускает HTTP-сервер на порту 80 для управления файлами на microSD через браузер.

### Токен авторизации

Каждая сессия генерирует случайный 4-символьный hex-токен (напр. `a3f1`). Полный URL с токеном отображается на экране Flipper'а:

```
http://192.168.1.42/?t=a3f1
```

Все HTTP-запросы требуют параметр `?t=XXXX`. Без токена — ошибка `403 Forbidden`. Токен автоматически включён во все ссылки веб-интерфейса.

### Операции

- **Просмотр** каталогов на SD-карте
- **Скачивание** любого файла
- **Загрузка** файлов через веб-форму
- **Создание** новых папок
- **Удаление** файлов и папок (с подтверждением)

### Как использовать

1. Откройте **Tools -> File Manager**
2. Flipper получает IP по DHCP, запускает HTTP-сервер
3. URL с токеном показан на экране
4. Откройте URL в любом браузере в той же сети
5. Нажмите Back на Flipper для остановки

### Совместимость

Работает в любом современном браузере (Chrome, Firefox, Safari, Edge). Тёмная тема, адаптивный дизайн для ПК и мобильных.

## ETH Bridge

ETH Bridge превращает Flipper в USB-Ethernet адаптер через протокол CDC-ECM.

### Как это работает

1. Откройте **Tools -> ETH Bridge**
2. Flipper переключает USB в режим CDC-ECM
3. Подключите телефон/ПК по USB — появится сетевой интерфейс
4. Весь трафик проходит через мост на уровне L2 между USB и Ethernet (W5500)
5. Хост получает IP от DHCP-сервера сети прозрачно

### Запись PCAP

Нажмите **OK** во время работы моста для старта/остановки записи трафика в `.pcap` файл на SD-карте. Файлы сохраняются в `apps_data/eth_tester/pcap/` с метками времени, совместимы с Wireshark.

### Совместимость с ОС

- **Linux**: нативная поддержка CDC-ECM, работает из коробки
- **macOS**: нативная поддержка CDC-ECM
- **Android**: работает на большинстве устройств с USB OTG
- **Windows**: может потребоваться RNDIS-драйвер

Нажмите **Back** для остановки и восстановления USB-профиля (CDC Serial для CLI Flipper'а).

## Технические детали

- **W5500 MACRAW**: Socket 0 с `MFEN=0` (принимает все фреймы, включая multicast)
- **Worker thread**: 8КБ стек, неблокирующий UI через ViewDispatcher
- **Кеширование DHCP**: одна сессия, результат переиспользуется всеми операциями
- **Безопасность памяти**: проверка границ во всех парсерах, буферы в куче
- **Уникальный MAC**: случайный MAC при первом запуске (аппаратный ГСЧ), сохраняется на SD. Два устройства не получат одинаковый MAC.
- **Порядок байтов**: ручной парсинг big-endian — без `htons`/`ntohs`

## Безопасность

Реализованы следующие меры защиты:

- **Защита от path traversal**: все URL-пути в File Manager и имена файлов TFTP проверяются — последовательности `..` отклоняются
- **Защита от XSS**: имена файлов экранируются (HTML entities) при отображении в веб-интерфейсе
- **Токен авторизации**: File Manager требует случайный сессионный токен во всех HTTP-запросах; токен отображается на экране Flipper'а
- **Санитизация загрузок**: из имён загружаемых файлов удаляются разделители путей (`/`, `\`)
- **Валидация DNS**: DNS-ответы проверяются на соответствие IP ожидаемого сервера
- **Защита mDNS**: рекурсивное следование DNS-указателям ограничено глубиной 4
- **Content-Disposition**: спецсимволы удаляются из имён файлов в HTTP-заголовках

## Ограничения

Невозможно реализовать на Flipper Zero + W5500:

- **802.1X** — нужен полноценный supplicant, не хватит RAM
- **Полный захват на 100 Мбит** — SPI ограничивает; PCAP в режиме Bridge записывает реально проходящий трафик
- **SNMP** — ASN.1 парсер слишком тяжёл для RAM
- **TLS/HTTPS** — нет криптобиблиотек в FAP SDK

## Решение проблем

| Проблема | Решение |
|----------|---------|
| "W5500 Not Found!" | Проверьте SPI-подключение. Убедитесь в подключении 3V3 и GND. |
| "No Link!" | Подключите Ethernet-кабель. Проверьте кабель и удалённый порт. |
| "DHCP failed" | Убедитесь в наличии DHCP-сервера в сети. Сначала проверьте Link Info. |
| Flipper зависает на Back | Исправлено в v1.0 — HTTP-сокет принудительно закрывается. Обновитесь. |
| File Manager показывает 403 | Укажите токен в URL: `?t=XXXX` (показан на экране Flipper'а). |
| PXE-клиент не загружается | Проверьте настройки сетевой загрузки в BIOS. Убедитесь что boot-файл в `apps_data/eth_tester/pxe/`. |
